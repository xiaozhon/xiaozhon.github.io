RESEARCH

Wireless Sensor Networks

  • WSN Deployment

    Developed applications for an out-door heterogeneous WSN testbed for hydrology and WSN research.
    -  The testbed has about 100 nodes of different types (i.e., MicaZ, IRIS, TelosB).
    -  Perform vairous tasks including upward routing, downward routing, over-the-air programming, compressed sensing, flash accessing, etc.
    -  Everything has to fit in the tiny nodes of 4 KB RAM (MicaZ) or 48 KB ROM (TelosB).

  • Downward Routing in IoT/WSNs

    Designed a reliable (>98%), energy efficient (negligible overhead), and extremely scalable downward routing protocol for IoT/WSN.
    -  Introduce opportunistic routing into source routing.
    -  Encode source-route using an adaptive Bloom filter.
    -  Build downward routing table based on upward traffic.
    -  Implemented in TinyOS/nesC for various sensor platforms.

  • MobileDeluge

    Designed a mobile over-the-air programming tool to assist the node reprogramming process in out-door, heterogeneous, and duty-cycled WSN deployments. Significantly reduced the labor for network maintenance.
    -  A control layer on top of exsiting approaches.
    -  A mobile command/query system that enables efficient onsite diagnosis.
    -  Implemented in TinyOS/nesC for various sensor platforms;
    -  Developed PC side gateway in Java.

  • Network Dynamics and WSN Benchmark

    Perform network analysis and devise a benchmark data suite for the research community.
    -  Network dynamics includes the link level characteristics, network level (topological) characteristics, and temporal characteristics.
    -  Measure the network level dynamics based on the number of links, node entropy, and link entropy for the network topology within every 45 minutes.
    -  The benchmark data suite includes link level information and full topology information.

  • Routing Topology Recovery

    Routing topology recovery algorithm based on compressed sensing for very dynamic WSNs.
    -  Encode the path information in 4-byte field in each upward packet.
    -  Implemented in TinyOS/nesC for various sensor platforms.
    -  Implemented PC side decompression algorithm in Python.

  • Sensor Boards for TelosB Motes

    Designed a sensor board for TelosB mote to drive external sensors.
    -  Sensor board is designed using Eagle.
    -  Sensor driver is developed based on TinyOS/nesC.
    -  For both analog and digital sensors, e.g., Decagon EC-5 soil moisture sensor, MSP-1 water potential sensor, and MPS-2 digital water potential sensor.
    -  Solved the clock drift problem (when temperature changes) of TelosB using a fridge and oscilloscope.

  • GureenGame

    Quality of Service control based on Gur Game to autonomously control the number of reports from the WSNs.

Internet of Things

  • Smart Home

    -  Use Raspberry Pi/Arduino to control home device with CoAP protocol.
    -  Develop Python server to send notifications using email (SMTP) and SMS (Twillio).

  • Green House

    -  Use TelosB platform (with Contiki OS) to monitor temperature, humidity, light, and soil moisture for home plants.
    -  Nodes communicates with the base station using RPL protocol.
